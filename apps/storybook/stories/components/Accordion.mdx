# Accordion

## Used to show and hide sections of related content on a page

[Ark UI](https://ark-ui.com/docs/components/accordion)

<Canvas of={AccordionStories.Basic} />

## Anatomy

**Note:** The `id` prop is required for accessibility. Each accordion must have a unique ID to ensure they have ARIA relationships. When multiple accordions appear on the same page, each one should have a unique item `title` to comply by with WCAG landmark requirements.

**Simple Usage:**

```tsx
<script>
  import { Accordion } from '$saas/accordion';

  const items = [
    { value: 'a', title: 'First Item', content: 'Some value 1...' },
    { value: 'b', title: 'Second Item', content: 'Some value 2...' },
    { value: 'c', title: 'Third Item', content: 'Some value 3...' },
  ];
</script>

<Accordion.Root id="faq-accordion" {items} collapsible defaultValue={['b']} />
```

**Composition API:**

```tsx
<script>
  import { Accordion } from '$saas/accordion';
</script>

<Accordion.Root id="settings-accordion" collapsible defaultValue={['b']}>
  <Accordion.Item value="a">
    <Accordion.ItemTrigger>First Item</Accordion.ItemTrigger>
    <Accordion.ItemContent>Some value 1...</Accordion.ItemContent>
  </Accordion.Item>

  <Accordion.Item value="b">
    <Accordion.ItemTrigger>Second Item</Accordion.ItemTrigger>
    <Accordion.ItemContent>Some value 2...</Accordion.ItemContent>
  </Accordion.Item>

  <Accordion.Item value="c">
    <Accordion.ItemTrigger>Third Item</Accordion.ItemTrigger>
    <Accordion.ItemContent>Some value 3...</Accordion.ItemContent>
  </Accordion.Item>
</Accordion.Root>
```
